# HTTP 与 JSON 基础

## 前端和后端怎么通信

前端和后端通过 HTTP 请求通信。前端发送请求，后端返回 JSON。

例子：

```text
GET /api/tasks
POST /api/samples
```

## 常见 HTTP 方法

| 方法 | 作用 | 项目例子 |
| --- | --- | --- |
| GET | 查询数据 | 查询任务列表 |
| POST | 新增或提交操作 | 新建任务、登录 |
| PUT | 修改数据 | 修改用户 |
| DELETE | 删除数据 | 删除用户 |

## JSON 是什么

JSON 是前后端传输数据的格式：

```json
{
  "id": 1,
  "title": "港口近岸水质巡检",
  "status": "进行中"
}
```

Laravel 中返回 JSON：

```php
return response()->json($task);
```

前端中读取 JSON：

```ts
return response.json()
```

## 常见状态码

| 状态码 | 含义 | 项目例子 |
| --- | --- | --- |
| 200 | 成功 | 查询成功 |
| 201 | 创建成功 | 新建任务成功 |
| 204 | 成功但无内容 | 删除用户成功 |
| 401 | 未登录 | 没带 token |
| 403 | 无权限 | 普通用户访问用户管理 |
| 422 | 参数错误 | 表单字段不合法 |

## 答辩说法

> 前端通过 HTTP 请求访问后端 API，后端用 JSON 返回数据。不同状态码可以表示成功、未登录、无权限或参数错误。
