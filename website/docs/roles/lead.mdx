## 用户管理怎么讲

你新增/明确负责用户管理这一块。

要看文件：

```text
backend/app/Http/Controllers/UserController.php
backend/app/Http/Controllers/AuthController.php
backend/app/Models/User.php
backend/tests/Feature/AuthUserManagementTest.php
```

必背说法：

> 用户管理不是完整 RBAC，只区分 `admin` 和 `user`。后端会根据 token 找到当前用户，再判断是否是管理员。只有管理员才能查询、创建、修改和删除用户。为了避免系统没人能管理，后端还限制不能删除当前登录用户、不能删除最后一个管理员，也不能把最后一个管理员降级。

## 你的必背说法

> 我负责 PHP 后端总体架构、API 路由入口、登录中间件、用户管理和接口集成。项目使用 Laravel 的路由、中间件、控制器、模型和迁移组织后端代码；前端通过 `api.ts` 请求 `/api/*` 接口，后端校验后写入 SQLite，再返回 JSON 刷新页面。具体业务控制器由对应同学负责，我不会替其他组员答辩。
