# 项目概览

## 一句话介绍

海洋环境巡检课程设计系统是一个前后端分离的简化业务系统，用于演示「巡检任务、样本登记、检测结果、异常处理、分析建议、首页统计」这一条完整业务链。

## 项目背景

海洋环境巡检通常需要记录巡检地点、采样时间、检测指标和异常情况。真实企业系统会包含权限、审计、队列、图像识别和复杂报表。本课程设计只保留最能说明业务闭环的部分，让系统适合本科生开发、理解和答辩。

## 建设目标

- 建立一个可以本地运行的 Laravel + React 项目。
- 使用 SQLite 保存演示数据，降低数据库配置成本。
- 用简单页面完成主要业务操作。
- 用清晰文档帮助队友理解系统并准备答辩。
- 保持代码结构直接，不引入企业级复杂架构。

## 用户角色

本系统不设计完整 RBAC 权限模型。答辩时可以按以下简化角色解释：

| 角色 | 主要操作 |
| --- | --- |
| 巡检人员 | 创建任务、登记样本、录入检测结果 |
| 处理人员 | 查看异常、填写处理结果 |
| 管理/答辩演示人员 | 查看首页统计、演示完整流程 |

## 业务闭环

```text
巡检任务 -> 样本登记 -> 检测结果录入 -> 异常上报处理 -> 简单分析建议 -> 首页统计
```

这条链路可以对应答辩讲解：

- **巡检任务：** 明确一次巡检的地点、时间、负责人和状态。
- **样本登记：** 把采样点、采样时间和样本编号记录下来。
- **检测结果：** 录入 pH、盐度、温度、溶解氧等指标。
- **异常处理：** 当检测值超过阈值时，生成或上报异常。
- **分析建议：** 根据异常类型给出简单建议，例如复检、重点关注某片海域。
- **首页统计：** 汇总任务、样本、异常和分析结果，便于展示系统价值。

## 项目边界

本项目是课程设计版本，不追求生产级能力。以下内容只在答辩中作为「可扩展方向」说明，不纳入当前版本：

- 复杂权限管理和组织架构。
- 大规模数据报表和 BI 看板。
- Redis 队列、消息通知、异步任务调度。
- 图片上传、遥感图像识别或 AI 模型训练。
- Docker 化生产部署和云原生运维。

## 答辩价值

本系统适合从 4 个角度答辩：

- **业务完整：** 有从任务到统计的闭环，不只是静态页面。
- **技术清楚：** Laravel 负责 API 和数据，React 负责页面交互。
- **数据可讲：** 表之间有明确关联，接口和页面能对应起来。
- **演示稳定：** SQLite 本地数据库方便重置和演示。
