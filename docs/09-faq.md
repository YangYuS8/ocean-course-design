# 常见问题

## 项目类问题

### 这个系统解决什么问题？

它解决海洋环境巡检过程中的信息记录和流程管理问题。系统把巡检任务、样本、检测结果、异常处理和分析建议串成一条业务链，方便展示巡检数据如何产生、处理和汇总。

### 为什么说这是简化版？

真实企业系统通常有权限、审计、队列、图像识别和复杂报表。本项目面向课程设计，只保留主流程，保证团队能开发、能运行、能答辩。

### 和原始 ocean 项目是什么关系？

本项目是新的课程设计版本，只延续海洋业务主题和 Laravel 技术方向，不修改原始项目，也不复制企业级复杂架构。

## 技术类问题

### 为什么后端使用 Laravel？

Laravel 是成熟的 PHP Web 框架，路由、Controller、Model、Migration 和 Seeder 结构清晰，适合课程设计展示后端工程化能力。

### 为什么前端使用 React + Vite？

React 适合构建组件化页面，Vite 启动快、配置轻，适合实现前后端分离的后台管理界面。

### 为什么默认使用 SQLite？

SQLite 不需要单独启动数据库服务，只需要一个本地文件，适合课堂演示和队友快速运行。后续如果需要，也可以迁移到 MySQL。

### 为什么不用 Redis 队列？

本项目没有耗时任务和高并发需求。加入 Redis 会增加部署和讲解成本，不符合课程设计的简化目标。

### 为什么不做完整 RBAC？

完整 RBAC 需要权限表、角色表、授权逻辑和管理页面，会分散重点。本项目更关注巡检业务闭环，因此只保留简单用户概念。

## 业务类问题

### 检测结果如何判断异常？

课程设计版本使用简单阈值规则。例如 pH 超出正常范围、溶解氧偏低或浊度过高时，可以标记为异常并生成异常记录。

### 分析建议如何生成？

分析建议根据检测结果和异常记录生成。当前版本是规则文本，例如建议复检、增加采样点、关注某片区域。这样可解释性强，适合答辩。

### 一个任务可以有多个样本吗？

可以。`inspection_tasks` 和 `samples` 是一对多关系，一个巡检任务下可以登记多个采样点或多个时间点的样本。

### 一个样本可以有多个检测结果吗？

可以。课程设计中可以理解为一个样本多次检测或复检。这样也方便演示异常变化。

## 运行类问题

### 后端启动失败怎么办？

先检查依赖和环境：

```bash
cd backend
composer install
php artisan key:generate
touch database/database.sqlite
php artisan migrate --seed
php artisan serve
```

如果仍失败，查看终端报错，通常是 PHP 版本、扩展或 `.env` 配置问题。

### 前端启动失败怎么办？

先重新安装依赖：

```bash
cd frontend
pnpm install
pnpm run dev
```

如果提示 `pnpm` 不存在，需要先安装 pnpm，或临时使用 npm 运行对应命令。

### 前端页面打开但没有数据怎么办？

检查 3 件事：

- 后端 `php artisan serve` 是否启动。
- 前端配置的 API 地址是否指向 `http://127.0.0.1:8000/api`。
- 后端是否执行过 `php artisan migrate --seed`。

### 演示数据乱了怎么办？

在后端目录执行：

```bash
php artisan migrate:fresh --seed
```

这会重建数据库并恢复演示数据。

## 答辩类问题

### 老师问系统亮点怎么回答？

可以回答：系统亮点是围绕海洋巡检建立了完整业务闭环，采用前后端分离架构，数据库关系清楚，默认 SQLite 便于演示，并用规则分析实现可解释的异常建议。

### 老师问不足怎么回答？

可以回答：当前版本是课程设计系统，没有做完整权限、复杂报表、异步任务和 AI 识别。后续可以扩展为多角色协作、地图展示、趋势图分析和智能预警。

### 老师问数据库设计怎么回答？

可以回答：核心表包括任务、样本、检测结果、异常和分析建议。任务与样本是一对多，样本与检测结果是一对多，样本也关联异常和分析建议。这些关系对应业务流程。

### 老师问前后端如何通信怎么回答？

可以回答：前端通过 HTTP 请求调用后端 `/api/*` 接口，传输格式是 JSON。后端用 Laravel Controller 接收请求，使用 Eloquent 操作 SQLite，然后返回 JSON 给前端展示。

### 老师问为什么没有复杂算法怎么回答？

可以回答：本项目重点是课程设计中的业务流程和系统实现。规则分析足够展示异常判断和建议生成，而且结果可解释。复杂算法可以作为后续扩展方向。
