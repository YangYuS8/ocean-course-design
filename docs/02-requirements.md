# 需求说明

## 功能需求

### 1. 首页统计

首页用于展示系统整体情况，适合作为答辩开场页面。

- 展示巡检任务总数、进行中任务数、样本总数、异常总数。
- 展示最近异常和最近分析建议，能看到分析报告摘要。
- 展示业务链说明，帮助观众理解系统流程。

### 2. 巡检任务管理

巡检任务表示一次海洋环境检查活动。

- 创建任务：填写任务名称、巡检区域、负责人、计划日期。
- 查看任务列表：展示状态、区域、负责人、样本数量。
- 修改任务：修正任务名称、区域、负责人、计划日期和任务说明，避免录入错误后只能删除重建。
- 开始任务：将任务状态从待开始改为进行中。
- 提交任务：将任务状态改为已完成。
- 删除任务：删除不再需要的巡检任务；如果任务下已有样本，会同步删除对应的检测结果、异常和分析记录。

### 3. 样本管理

样本属于某一个巡检任务。

- 登记样本：填写样本编号、采样点、采样人、采样时间、现场天气、采样坐标和备注。
- 查看样本列表：按任务、状态或采样点查看。
- 查看样本详情：展示样本信息、检测结果、异常情况和分析报告摘要。

### 4. 检测结果录入

检测结果用于记录样本的关键指标。

- 为样本录入检测结果。
- 推荐指标：pH、盐度、温度、溶解氧、浊度。
- 根据简单阈值判断是否异常。
- 支持查看全部检测结果列表。

### 5. 异常上报与处理

异常用于记录检测结果中的风险情况。

- 自动或手动创建异常记录。
- 展示异常等级、异常指标、说明和状态。
- 处理异常：填写处理说明，将状态改为已处理。

### 6. 简单分析建议

分析建议用于课程答辩展示，不做复杂算法。

- 对某个样本发起分析。
- 根据检测值和异常记录生成文字建议。
- 将分析结果和报告摘要保存到分析记录表。
- 在样本详情和首页展示最近建议。

## 非功能需求

| 类别 | 要求 |
| --- | --- |
| 易运行 | 默认使用 SQLite，减少本地环境依赖 |
| 易理解 | 控制表数量和代码层次，避免过度封装 |
| 易演示 | 首页和演示流程能清楚展示业务闭环 |
| 易维护 | 文档说明接口、表结构、运行方式和分工 |
| 中文友好 | 页面和文档默认使用中文 |

## 不做内容

为了保证课程设计可控，本项目不实现以下能力：

- 完整 RBAC 权限系统。
- 审计日志系统。
- Redis 队列和异步 Worker。
- Python 分析服务。
- 图片上传和图像识别。
- 复杂国际化。
- Docker 优先部署。

## 验收标准

课程设计验收时建议按以下标准检查：

- 后端可以完成数据库迁移和种子数据填充。
- 前端可以启动并访问主要页面。
- 能走通一条完整演示链路：任务、样本、结果、异常、分析、统计。
- 主要表结构、接口路径和页面功能能被队友解释清楚。
- 文档能支持新人按步骤运行项目。
