## Git 提交建议

提交信息尽量清楚，建议使用：

```text
docs: add course defense documentation
feat: add inspection task api
feat: add samples page
fix: correct sqlite database config
```

每次提交只包含一类变更。不要把后端、前端、文档和格式化大范围混在一个提交里。

## 答辩材料准备清单

- [ ] 项目背景和目标。

## 代码检查清单

后端：

- [ ] 迁移文件可以执行。
- [ ] Seeder 可以生成演示数据。
- [ ] API 返回 JSON。
- [ ] 关键字段有校验。
- [ ] 状态变化符合业务流程。

前端：

- [ ] 页面中文标签清楚。
- [ ] 表格和表单能正常使用。
- [ ] API 错误有基本提示。
- [ ] 首页统计能更新。
- [ ] 演示时不依赖临时数据。

文档：

- [ ] README 能指导运行。
- [ ] 数据/API 文档和实际实现一致。
- [ ] 演示脚本可以照着讲。
- [ ] FAQ 覆盖老师常问问题。
